# Accessibility Features - WCDMR 2026 Website

This website has been designed to be fully accessible for **deaf/hearing impaired** and **blind/visually impaired** users, following WCAG 2.1 AA standards.

## Features for Blind/Visually Impaired Users

### Screen Reader Support
- ✅ **ARIA labels and roles** throughout the site
- ✅ **Semantic HTML** with proper heading hierarchy
- ✅ **Screen reader announcements** for form errors and success messages
- ✅ **Live regions** for dynamic content updates
- ✅ **Alt text** for all images with descriptive information
- ✅ **Skip to main content** link for keyboard navigation

### Keyboard Navigation
- ✅ **Full keyboard accessibility** - all interactive elements are keyboard accessible
- ✅ **Visible focus indicators** - 3px gold outline on all focusable elements
- ✅ **Logical tab order** throughout the site
- ✅ **Minimum touch target size** - 44x44px for all buttons and links

### Form Accessibility
- ✅ **Proper label associations** - all form fields have associated labels
- ✅ **Required field indicators** - visual and screen reader announcements
- ✅ **Error announcements** - screen readers announce validation errors
- ✅ **Fieldset and legend** for grouped form controls
- ✅ **aria-required** and **aria-invalid** attributes
- ✅ **aria-describedby** linking fields to error messages

### Visual Accessibility
- ✅ **High color contrast** - meets WCAG AA standards (4.5:1 for text)
- ✅ **Focus indicators** - 3px gold outline on all interactive elements
- ✅ **Text resizing support** - text can be resized up to 200% without loss of functionality
- ✅ **Reduced motion support** - respects prefers-reduced-motion preference
- ✅ **High contrast mode support** - adapts to system high contrast settings

## Features for Deaf/Hearing Impaired Users

### Visual Communication
- ✅ **All information is text-based** - no audio-only content
- ✅ **Clear visual indicators** for all actions and states
- ✅ **Visual error messages** with icons and clear text
- ✅ **Visual success confirmations** with checkmarks and messages
- ✅ **Social media links** clearly labeled and visible

### Form Communication
- ✅ **Clear visual form labels** - all fields clearly labeled
- ✅ **Visual validation feedback** - errors shown with red text and warning icons
- ✅ **Visual success messages** - registration success clearly displayed
- ✅ **Visual payment confirmation** - payment status clearly shown

## Testing with Assistive Technologies

### Screen Readers
Test with:
- **NVDA** (Windows, free)
- **JAWS** (Windows, paid)
- **VoiceOver** (Mac/iOS, built-in)
- **TalkBack** (Android, built-in)

### Keyboard Navigation
- Use **Tab** to navigate forward
- Use **Shift+Tab** to navigate backward
- Use **Enter** or **Space** to activate buttons/links
- Use **Arrow keys** for radio buttons and checkboxes

### Browser Zoom
- Test at **200% zoom** - all content should remain accessible
- Text should remain readable and functional
